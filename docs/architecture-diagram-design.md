## L 型包裹结构详解（Biz 色块向上延伸）

```
┌──────────────┬────────────────────────┐
│              │  Biz 色块向上延伸 →     │
│  Agent 块    │  ┌─────┐ ┌─────┐      │
│              │  │📊   │ │👔   │      │
│  ┌────┐┌───┐ │  │状态机│ │审批链│      │
│  │🔧22││🔄  │ │  └─────┘ └─────┘      │
│  └────┘└───┘ │  ┌─────┐ ┌─────┐      │
│  ┌────┐      │  │💰   │ │✅   │      │
│  │🤖  │      │  │预算 │ │合规 │      │
│  └────┘      │  └─────┘ └─────┘      │
├──────────────┴────────────────────────┤
│      ⚙️ 业务逻辑层 Biz（下方全域）      │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐     │
│  │👥   │ │🏢   │ │📧   │ │📄   │     │
│  │员工 │ │部门 │ │邮件 │ │PDF  │     │
│  └─────┘ └─────┘ └─────┘ └─────┘     │
└───────────────────────────────────────┘
```

**关键**：Biz 层色块由两部分组成——**下方横条**（全宽） + **右侧纵条**（向上延伸至 Agent 侧边），形成 L 型。
Agent 层嵌入 L 型的左上缺口中，视觉上被 Biz "半包裹"。

## 各层元素定义

### 1. 前端层
- **标题**：`🌐 前端层 Frontend — React 19 + Ant Design 6 + Zustand 5`
- **子元素**（4 个等宽方块，仅图标无文字）：
  - `🧠` — ThinkingCard
  - `🔧` — ToolCard  
  - `💬` — MessageCardView
  - `⚠️` — InterruptBar
- **底部横条**：`SSE Stream · REST API`（仅标题）

### 2. 服务层
- **标题**：`🔌 服务层 Service / HTTP — Gin`
- **子元素**（8 个缩写标签，等宽横排）：
  `Auth` `Agent` `Emp` `Dept` `Bgt` `Rmb` `Apr` `Cmp`

### 3. Agent 层 ★（视觉重心）
- **标题**：`🧠 智能体层 Agent — Blades 框架`
- **子元素**（4 个方块，图标+双字标签，无描述）：
  - `🔧` `22 工具` — 完整工具体系
  - `🔄` `ReAct` — 推理-行动循环
  - `⏸️` `中断` — 两阶段确认
  - `🤖` `子 Agent` — 合规审核子Agent
- **边框**：3px 加粗 `#7A46FF`
- **高度**：占总高度的 ~20%

### 4. Biz 层（L 型包裹 Agent）
- **标题**：`⚙️ 业务逻辑层 Biz — Domain Services`
- **子元素 8 个**（4+4 两行横排，图标+双字标签）：
  - 第一行：`📊 状态机` `👔 审批链` `💰 预算` `✅ 合规`
  - 第二行：`👥 员工` `🏢 部门` `📧 邮件` `📄 PDF`
- **L 型结构**：主体在 Agent 正下方，右翼延伸至与 Agent 等宽

### 5. 基础设施层
- **标题**：`🧱 基础设施层 Infra — 全配置可插拔`
- **子元素**（8 个缩写标签）：
  `Emb` `VS` `OCR` `Stor` `Ses` `PDF` `Mail` `SMTP`

### 6. 数据层
- **标题**：`💾 数据层 Data`
- **子元素**（4 个名称标签，无描述）：
  `MySQL 8` `Redis` `MinIO` `pgvector`

## 左侧贯穿区域（竖条，宽 ~90px）

从上到下排列 6 个标签方块，与右侧主层同一水平线：

| 标签 | 含义 | 图标 |
|------|------|------|
| `Wire DI` | 编译时依赖注入 | 🔌 |
| `Viper` | 配置管理 | ⚙️ |
| `Zap` | 结构化日志 | 📝 |
| `JWT·RBAC` | 认证授权 | 🔐 |
| `CORS` | 跨域中间件 | 🌍 |
| `Log` | 请求日志 | 📋 |

每个为小圆角矩形，左对齐，贯穿整个右侧区域高度。

---

## 层间连接

- **前端 → 服务**：虚线箭头 `↓`
- **服务 → Agent**：虚线箭头 `↓`
- **Agent → Biz**：实线 L 型箭头（包裹关系）
- **Biz → 基础设施**：虚线箭头 `↓`
- **基础设施 → 数据**：虚线箭头 `↓`
- **左侧贯穿 → 各层**：水平虚线（表示横切关注点）
