AGENT DESIGN

Agent 两大设计模式

🧩 装饰器模式

RoleGuard — 运行时校验角色，fail-closed
Interruptable — 首次调用暂停，用户确认后执行
应用于 4 个敏感工具：提交 · 审批 · 取消
任意工具 + 装饰器 = 鉴权 + 确认 — 零代码改动

🤖 子 Agent 嵌套

独立 ReAct 循环 — 子Agent 自己思考→检索→判定
主Agent 无感知 — 只看干净 JSON 判断结果
同时有规则引擎（关键词+阈值）双路径保障
主Agent 只需关心结果 — 关注点分离 · Agent 可组合
设计哲学：关注点分离 · 零侵入扩展 · Agent 组合复用 · 人机协同确认

INTERRUPT

两阶段中断确认机制

❌ 无确认机制

LLM 直接调用 → 理解偏差 → 误操作 → 不可逆损失

✅ 两阶段确认

暂停 → 用户确认 → 重放工具 → 继续执行 · 单次有效防重放

关键状态字段
ApprovalState { Approved, Reason, Consumed }
session.State["pending_tool"]
session.State["approval"]
