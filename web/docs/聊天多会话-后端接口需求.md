## 二、缺失的 HTTP 接口（需新增）

### 2.1 GET /api/chat/sessions

获取当前登录用户的会话列表。

| 项 | 值 |
|------|-----|
| 方法 | `GET` |
| 路径 | `/api/chat/sessions` |
| 认证 | Bearer JWT（中间件 `AuthMiddleWire`） |
| 路由位置 | `internal/router/root.go` — `api` 组内 |
| 查询参数 | 无（用户身份从 JWT metadata 提取） |

**成功响应**（200）：

```json
[
  {
    "session_id": "018f4a3c-...",
    "summary": "我要报销上个月去北京出差的差旅费",
    "message_count": 12,
    "status": "active",
    "created_at": "2026-07-05T10:30:00Z",
    "updated_at": "2026-07-08T14:22:00Z"
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `session_id` | string | 会话 UUID v7 |
| `summary` | string | 会话摘要（第一条用户消息的前 50 字） |
| `message_count` | uint | 消息总数 |
| `status` | string | `active` / `completed` / `expired` |
| `created_at` | string | 创建时间（ISO 8601） |
| `updated_at` | string | 最后活跃时间（ISO 8601） |

**错误响应**：

| 状态码 | 说明 |
|--------|------|
| 401 | 未认证 |
| 500 | 查询失败 |

**实现要点**：

- 调用 `s.store.ListSessions(ctx, meta.UserID, "")`（status 空字符串≈查询全部）
- 返回时过滤掉 `status = "expired"` 或由前端过滤
- 仅返回当前登录用户自己的会话（不能看到别人的）
- 按 `updated_at DESC` 排序（infra 层已实现）

### 2.2 DELETE /api/chat/sessions/:id

删除（软删除）指定会话。

| 项 | 值 |
|------|-----|
| 方法 | `DELETE` |
| 路径 | `/api/chat/sessions/:id` |
| 认证 | Bearer JWT |
| 路由位置 | `internal/router/root.go` — `api` 组内 |
| 路径参数 | `id` — 会话 UUID |

**成功响应**（200）：

```json
{ "ok": true }
```

**错误响应**：

| 状态码 | 说明 |
|--------|------|
| 400 | 缺少 session_id |
| 401 | 未认证 |
| 404 | 会话不存在或不属于当前用户 |
| 500 | 删除失败 |

**实现要点**：

- 先调用 `GetSession` 校验 session 存在且 `UserID == 当前用户ID`
- 调用 `s.store.UpdateSession(ctx, sessionID, map[string]any{"status": "expired"})` 做软删除
- 不删除消息数据（保留审计追溯），仅标记 expured

### 2.3 GET /api/chat/sessions/:id/messages

分页加载会话历史消息（游标分页，供前端懒加载用）。

| 项 | 值 |
|------|-----|
| 方法 | `GET` |
| 路径 | `/api/chat/sessions/:id/messages` |
| 认证 | Bearer JWT |
| 路径参数 | `id` — 会话 UUID |
| 查询参数 | `before_seq` — uint，可选。不传返回最新 N 条，传了返回 seq 小于该值的更早 N 条 |

**成功响应**（200）：

```json
{
  "messages": [
    {
      "seq": 1,
      "role": "user",
      "content": "我想报销一张发票",
      "tool_name": "",
      "created_at": "2026-07-05T10:30:00Z"
    },
    {
      "seq": 2,
      "role": "assistant",
      "content": "好的，请上传您的发票图片...",
      "tool_name": "",
      "created_at": "2026-07-05T10:30:05Z"
    }
  ],
  "has_more": true,
  "first_seq": 1
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `messages` | array | 消息列表（按 seq 正序） |
| `messages[].seq` | uint | 消息在会话内的序号 |
| `messages[].role` | string | `user` / `assistant` / `tool` / `system` |
| `messages[].content` | string | 消息文本内容 |
| `messages[].tool_name` | string | 工具名（仅 tool 角色时有值） |
| `messages[].created_at` | string | 创建时间 |
| `has_more` | bool | 是否还有更早的消息 |
| `first_seq` | uint | 本批消息中的最小 seq（用于下次请求的 before_seq） |

**分页示例**：

```
首次加载（最新 20 条）:
  GET /api/chat/sessions/:id/messages
  → { messages: [seq 81..100], has_more: true, first_seq: 81 }

用户滚动到顶加载更早:
  GET /api/chat/sessions/:id/messages?before_seq=81
  → { messages: [seq 61..80], has_more: true, first_seq: 61 }

再次滚动:
  GET /api/chat/sessions/:id/messages?before_seq=61
  → { messages: [seq 1..60], has_more: false, first_seq: 1 }
```

**错误响应**：

| 状态码 | 说明 |
|--------|------|
| 400 | 缺少 session_id |
| 401 | 未认证 |
| 404 | 会话不存在或不属于当前用户 |
| 500 | 查询失败 |

**实现要点**：

- 每页固定返回 20 条
- 必须先校验 session 归属（`GetSession` 检查 UserID）
- `before_seq` 为空时调用现有 `GetHistory(ctx, sessionID, 20)`
- `before_seq` 有值时调用新增的 `GetHistoryBefore(ctx, sessionID, beforeSeq, 20)`

## 三、缺失的 infra 方法（需新增）

### 3.1 SessionStore.GetHistoryBefore

在 `infra/session.go` 接口中新增方法定义，在 `infra/session_mysql.go` 中实现。

```go
// GetHistoryBefore 获取指定 seq 之前的 N 条历史消息（倒序，用于游标分页）
// beforeSeq: 游标，返回 seq 严格小于此值的消息
// limit: 返回条数
// 返回的消息按 seq 正序排列（便于前端直接追加到列表头部）
GetHistoryBefore(ctx context.Context, sessionID string, beforeSeq uint, limit int) ([]*schema.Message, error)
```

**MySQL 实现**：

```go
func (s *MySQLSessionStore) GetHistoryBefore(ctx context.Context, sessionID string, beforeSeq uint, limit int) ([]*schema.Message, error) {
    var records []model.SessionMessage
    if err := s.db.WithContext(ctx).Raw(
        "SELECT * FROM (SELECT * FROM session_messages WHERE session_id = ? AND seq < ? ORDER BY seq DESC LIMIT ?) AS older ORDER BY seq ASC",
        sessionID, beforeSeq, limit,
    ).Scan(&records).Error; err != nil {
        return nil, fmt.Errorf("查询会话更早历史失败: %w", err)
    }

    msgs := make([]*schema.Message, 0, len(records))
    for _, rec := range records {
        msg := s.restoreMessage(&rec)
        if msg != nil {
            msgs = append(msgs, msg)
        }
    }
    return msgs, nil
}
```

**Redis 缓存策略**：由于懒加载请求的是不同 seq 区间的数据，不适合缓存全量。`GetHistoryBefore` 不读缓存，直接从 MySQL 查询。
